# CODE D'ERREUR D'INTERRUPTION
tous les interrupt entre 0 et 32 sont des interruptions d'erreur
certains code d'erreur peuvent être corrigé après le retour de l'interruption et d'autres non
| Code de l'interruption | Nom                                  | Descriptions                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  | Contient un  code d'erreur ?     |
|------------------------|--------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------|
| 0                      | Division par 0                       | Cette exceptions est produites  quand l'instruction DIV/IDIV est utilisé avec un 0                                                                                                                                                                                                                                                                                                                                                                                                            | non                              |
| 1                      | Debug                                | Cette erreur __intentionnelle__ est  généralement pour le débogage                                                                                                                                                                                                                                                                                                                                                                                                                            | non                              |
| 2                      | Interruption nmi                     | L'interruption nmi est une interruption causé par des éléments externe comme la ram                                                                                                                                                                                                                                                                                                                                                                                                           | non                              |
| 3                      | Breakpoint                           | Cette erreur __intentionnelle__ est utilisé pour le débogage                                                                                                                                                                                                                                                                                                                                                                                                                                  | non                              |
| 4                      | Dépassement                          | L'interruption 4 est causé quand l'instruction `INTO` est éxécuté alors que le bit 11 de RFLAGS est mit à 1 note: l'erreur n'est pas possible en 64 bit car l'instruction `INTO` n'est pas disponible en long  mode                                                                                                                                                                                                                                                                           | non                              |
| 5                      | Dépassement de table                 | L'interruption 5 est causé quand l'instruction `BOUND` est exécuté quand opérateur 1 n'est pas dans la taille de table définis dans l'opérateur 2 note: l'erreur n'est pas possible en 64 bit car l'instruction `BOUND` n'est pas disponible en long mode                                                                                                                                                                                                                                     | non                              |
| 6                      | Instruction  non valide              | L'interruption 6 est causé quand :  - on essaye d'accéder à un registre non existant  - on essaye d'exécuter un une instruction non disponible  - quand UD est exécuté                                                                                                                                                                                                                                                                                                                        | non                              |
| 7                      | Appareil non  disponible             | L'interruption 7 est appelé quand on essaye d'initialiser le fpu alors qu'il n'existe pas                                                                                                                                                                                                                                                                                                                                                                                                     | non                              |
| 8                      | Faute Double                         | La faute double est appelé quand il y a une erreur pendant que l'interruption d'erreur est appelé  (une erreur dans une erreur)                                                                                                                                                                                                                                                                                                                                                               | oui                              |
| 9                      | Erreur de  Segment de  coprocesseur  | Cette erreur n'est plus utilisé                                                                                                                                                                                                                                                                                                                                                                                                                                                               | non                              |
| 10                     | TSS invalide                         | L'interruption TSS invalide est exécuté quand le  sélecteur de segment pour la tss est invalide Causé pendant un changement de task ou pendant l'accès de la tss                                                                                                                                                                                                                                                                                                                              | oui (code  d'erreur de  segment) |
| 11                     | Segment non  présent                 | L'interruption segment non présent est exécuté quand  on essaye de charger un segment qui à le bit présent à 0                                                                                                                                                                                                                                                                                                                                                                                | oui (code  d'erreur de  segment) |
| 12                     | Segment de pile invalide             | L'interruption Segment de pile invalide est causé quand on charge un segment de stack qui n'est pas présent, - quand la vérification de la limite de pile n'est pas possible - quand (en 64bit) on essaye de faire une opération qui fait une référence la mémoire en utilisant le pointeur de stack (rsp) qui contient une adresse mémoire non canonique  - quand le segment de stack n'est pas présent pendant une opération qui fait référence au registre ss, (comme pop, push, iret ...) | oui (code  d'erreur de  segment) |
| 13                     | Fautes générale de protection        | L'interruption n°13 peut être causé pour beaucoup de raison, dont: - écrire un 1 dans une zone du registre CR4 réservé - utiliser une instruction SSE qui essaye d'accéder une zone de la mémoire 128 bit qui n'est pas aligné en 16bit  - la pile de mémoire n'est pas aligné en 16bit [...] voir le manuel intel pour plus d'information (chap 3 6.15.13)                                                                                                                                   | oui (code  d'erreur de  segment) |
| 14                     | Faute de page                        | L'interruption n°14 peut être causé quand il y a une erreur en relation avec le paging. Quand on essaye d'accéder à une zone de la mémoire qui n'a pas de table présente.  Quand on essaye de charger une table et que la zone ou on éxécute le code n'est pas exécutable dans la page  quand un problème d'autorisation est causé, (ex: écrire dans une zone de la mémoire qui ne peut pas être écrite)  [...] voir le manuel intel pour plus d'information (chap 3 6.15.14)                 | oui (code  d'erreur de page)     |
| 15                     | réservé                              | //                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            | non                              |
| 16                     | Faute du fpu x87                     | L'interruption n°16 est causé quand il y a une erreur pendant une instruction du fpu - Opération invalide  - division par 0  - Dépassement numérique  - Résultat non exact   - Le bit 5 du registre CR0 = 1                                                                                                                                                                                                                                                                                   | non                              |
| 17                     | Faute D'alignement                   | Pour qu'il y ait une faute d'alignement il faut que le bit 18 de CR0 et RFLAGS soient égaux à 1 L'erreur est causé quand une référence de mémoire est non aligné                                                                                                                                                                                                                                                                                                                              | oui                              |
| 18                     | Faute de vérification de machine     | Pour qu'il y ait une faute de vérification de machine il faut que le bit 6 du cr4 soit à 1.   L'erreur est causé quand le cpu détecte une erreur de machine, comme:  - un problème de bus, cache, mémoire, ou une erreur interne                                                                                                                                                                                                                                                              | non                              |
| 19                     | Exception de variable a virgule SIMD | L'interrupt n°19 est appelé quand il y a une érreur avec les nombres a virgules pendant une opération SSE - division par 0  - Dépassement numérique  - Résultat non exact  - [...]                                                                                                                                                                                                                                                                                                            | non                              |
| 20                     | Exception de virtualization          | L'exception de virtualization est appelé quand il y a une violation de droit avec une instruction EPT                                                                                                                                                                                                                                                                                                                                                                                         | non                              |
| 21 à 31                | réservé                              | //                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            | non                              |
| ---                    | Faute triple                         | L'exception faute triple est exécuté quand il y a une erreur pendant l'interruption de faute double (une erreur dans une erreur dans une erreur) l'interruption faute triple cause un redémarrage de la machine                                                                                                                                                                                                                                                                               | non                              |



## Code d'erreur d'une faute de page
| BIT  	| NOM 	| DESCRIPTION                                                                                                                                      	|
|------	|-----	|--------------------------------------------------------------------------------------------------------------------------------------------------	|
| 0    	| P   	| (p=1)<br>la faute a été causée à cause <br>d'une violation de protection<br><br>(p=0)<br>la faute a été causée<br>car la page n'est pas présente 	|
| 1    	| W   	| (W=0)<br>la faute a été causée à cause<br>d'une lecture<br>(W=1)<br>la faute à été causée à cause<br>d'une écriture                              	|
| 2    	| U   	| (U=1)<br>la faute a été causée car <br>la page n'est pas utilisateur<br>alors que CPL = 3                                                        	|
| 3    	| R   	| (R=1)<br>la faute a été causée car<br>la page contenait un bit<br>réservé                                                                        	|
| 4    	| I   	| (I=1)<br>la faute a été causée a cause<br>d'une lecture à cause d'une <br>instruction                                                            	|
| 5    	| PK  	| (PK=1) <br>la faute a été causée à cause<br>d'une violation de droit<br>de clé                                                                   	|
| 6    	| SS  	| (SS=1)<br>la faute a été causée à cause<br>d'un accès à << l'ombrede la stack >>                                                             	|
| 7-31 	| //  	| Réservé                                                                                                                                          	|

Lors d'une faute de page, l'addresse qui a causé l'exception est
stockée dans `CR2`

| 3   	| Index 	| 13     	| Index de la table sélectionné <br>dans TBL                                                                                                                         	|                                                                                                       	|
